## 🏗️ Общая архитектура проекта

**Стоматологическая поликлиника №22** — это интерактивный чат-бот на базе Flask с интеграцией Yandex GPT и Yandex SpeechKit для автоматизированного информирования пациентов. Проект предоставляет:

- 💬 Чат-интерфейс с возможностью текстового и голосового ввода
- 🔊 Автоматическую озвучку ответов с паузами между абзацами (SSML)
- 🎯 Контекстные подсказки и динамическое меню
- 🔐 Админ-панель для управления контентом
- 🧠 Базу знаний с поддержкой Markdown и автоответов
- 📞 Интеграцию с бронированием и калькулятором услуг

Проект развёрнут локально или на сервере и доступен по IP (`192.168.x.x`) или через туннель (ngrok).

## 🧱 Подробное описание каждого компонента

### 1. Ядро приложения

#### `web_app.py`
- **Назначение**: Основной файл приложения, запускающий Flask-сервер
- **Функции**:
  - Инициализация Flask-приложения и загрузка конфигурации
  - Загрузка переменных окружения из `.env`
  - Определение всех маршрутов API и страниц
  - Интеграция с Yandex GPT через REST API
  - Интеграция с Yandex SpeechKit (TTS и STT)
  - Управление пользовательскими сессиями и аутентификацией
  - Обработка запросов к базе знаний, бронированиям и подсказкам
  - Логирование всех взаимодействий и аудит действий

- **Ключевые функции**:
  - `call_yandex_gpt()` — генерация ответов через Yandex GPT
  - `load_knowledge_base()` — загрузка и кэширование базы знаний
  - `load_suggestion_map()` — загрузка контекстных подсказок
  - `log_interaction()` — запись диалогов в `interaction_log.json`
  - `load_menu()` — загрузка структуры меню
  - `save_*()` — сохранение изменений в JSON-файлы

- **🆕 Новые функции**:
  - `/tts` — эндпоинт для озвучки текста через Yandex SpeechKit с поддержкой SSML
  - `/voice` — обработка аудиозапросов (если используется серверное распознавание)
  - `prepare_text_for_tts()` — предобработка текста (версии, даты, числа)

### 2. Интерфейс пользователя

#### `templates/index.html`
- **Назначение**: Главная страница чат-бота

### 3. База знаний и контекстные данные

#### `data/knowledge_base.json`
- **Назначение**: Хранение быстрых ответов на часто задаваемые вопросы
- **Структура**: Объект "вопрос-ответ"
```json
{
  "график работы": "При очном посещении поликлиники: обязательная запись через информаты...<br><br>При дистанционной записи: запись по телефону: (495) 539 30 00"
}
```
- **Особенности**:
  - Поддерживает Markdown (`**жирный**`, `*курсив*`)
  - Приоритет над GPT — экономит API-запросы
  - Автоматически обновляется через админку

#### `data/suggestions.json`
- **Назначение**: Контекстные подсказки с готовыми ответами
- **Структура**: Объект с темами и массивами подсказок
```json
{
  "запись": [
    {
      "text": "Цены",
      "question": "стоимость записи",
      "answer": "Запись платная, от 500 ₽."
    }
  ]
}
```
- **Особенности**:
  - Группировка по темам для контекстного отображения
  - Загружаются через `/suggestions/all` API
  - Можно редактировать в админке

#### `data/menu.json`
- **Назначение**: Определение структуры главного меню
- **Структура**: Массив объектов с метаданными
```json
[
  {
    "admin_text": "Услуги",
    "display_text": "🦷 Услуги стоматологии",
    "question": "услуги",
    "category": "services",
    "price_info": "от 500 ₽",
    "suggestion_topic": "services"
  }
]
```
- **Особенности**:
  - Двойная система имён: `admin_text` (для админа), `display_text` (для пользователя)
  - Привязка к категориям из `menu_categories.json`
  - Поддержка PNG-иконок через `<img src="/static/icons/logo.png">`

#### `data/menu_categories.json`
- **Назначение**: Хранение категорий для организации меню
- **Структура**:
```json
{
  "services": "🛠️ Услуги",
  "info": "ℹ️ Информация",
  "booking": "📅 Запись"
}
```
- **Особенности**:
  - Плоская структура для простоты
  - Защита системных категорий от удаления
  - Автоматическое создание при первом запуске

### 4. Система хранения данных

#### `data/interaction_log.json`
- **Назначение**: История всех диалогов с пользователями
- **Структура**: Массив записей
```json
[
  {
    "timestamp": "2025-04-05T10:30:00",
    "question": "график работы",
    "answer": "Работаем с 8:00 до 20:00...",
    "source": "knowledge_base"
  }
]
```
- **Особенности**:
  - Отметка источника: `knowledge_base`, `suggestions`, `gpt`
  - Используется для анализа эффективности
  - Автоматическое резервное копирование

#### `data/feedback.json`
- **Назначение**: Хранение оценок пользователей (👍/👎)
- **Структура**:
```json
[
  {
    "timestamp": "2025-04-05T10:30:05",
    "question": "график работы",
    "rating": "good"
  }
]
```
- **Особенности**:
  - Помогает улучшать качество ответов
  - Интеграция с аналитикой

#### `data/booking.json`
- **Назначение**: Хранение заявок на запись
- **Структура**:
```json
[
  {
    "name": "Иван Петров",
    "phone": "+7 (999) 123-45-67",
    "date": "2025-04-10",
    "service": "Прием у терапевта",
    "status": "pending"
  }
]
```
- **Особенности**:
  - Валидация данных на клиенте и сервере
  - AJAX-отправка с обратной связью
  - Интеграция с админкой
